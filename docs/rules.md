# 협업 규칙 (v1)

## 기본 원칙
- 새 정보가 필요하면 반드시 먼저 질문한다.
- 명령어 실행은 사용자 승인 후 진행한다.
- 파일 변경은 요약 후 진행한다.
- 범위 밖 제안은 옵션으로만 제시한다.

## 진행/관리
- 단계 완료 시마다 “to‑do 리스트 갱신할까요?”를 먼저 묻는다.
- 작업 범위가 커지면 작은 단위로 쪼개어 합의 후 진행한다.

## 바이브 코딩 안전 수칙
- 작업 전: 목표 1개만 선언한다.
- 작업 중: 변경 로그 1줄을 기록한다.
- 작업 후: 결과/근거/다음 단계를 요약한다.
- 요청/프롬프트 범위를 크게 벗어나면 반드시 되물어 확인한다.

## 자동화/권한
- 사용자가 요청하지 않은 실행/설치는 하지 않는다.
- 임의로 도구/명령을 실행하지 않는다.

## 문서/정책
- 정책/스코프는 HackerOne 일반 정책 준수를 기본으로 한다.
- 타깃 다운로드는 공식 배포 채널 + 버전 고정 + 해시 기록을 기본으로 한다.
